🚀 Bitcoin Bastion Bot — умный Telegram-бот для криптоаналитики

Bitcoin Bastion Bot— это многофункциональный Telegram-бот, который в реальном времени отслеживает криптоновости, анализирует их тональность с помощью ML и определяет, какие из них спровоцировали значимое движение цены. Такие новости автоматически публикуются в канал. Бот также предоставляет пользователям широкий спектр рыночных метрик, умеет работать в группах и имеет удобную панель администратора.

Бот построен на базе бесплатного API free-crypto-news (также известного как Bitcoin-Bastion-News-API), который агрегирует новости из 200+ источников, включая международные, и предоставляет готовые ML-эндпоинты (тональность, суммаризация, ответы на вопросы).

✨ Возможности

· 📡 Реалтайм‑новости — подключение к SSE‑стриму API для получения новостей о Bitcoin и других криптовалютах.
· 🧠 ML‑анализ — для каждой новости запрашивается тональность (/api/ai/sentiment). Если тональность совпадает с направлением движения цены в течение заданного времени — новость помечается как «спровоцировавшая движение».
· 📢 Публикация в канал — только самые важные (триггерные) новости отправляются в указанный Telegram‑канал (раз в 30 минут или по расписанию).
· 📊 Рыночные метрики в личных чатах — команды для получения цены BTC, индекса страха и жадности, доминации, китовых транзакций, ликвидаций, фандинга и многого другого.
· 👥 Групповой чат — публикация всех свежих новостей в группу (с настраиваемым интервалом) для общего обсуждения.
· 🔔 Подписки — пользователи могут подписаться на уведомления о китах, ликвидациях или триггерных новостях (уведомления приходят в личку).
· 🛠 Панель администратора — настройка порогов срабатывания триггера, просмотр статистики, рассылка сообщений пользователям и публикация в канал вручную.
· 🌐 Веб‑дашборд — простая веб‑страница для мониторинга статистики (количество новостей, триггерных событий, публикаций).
· ⚡ Кэширование — Redis используется для снижения нагрузки на внешние API.

📦 Установка и запуск

Предварительные требования

· Установленный Python 3.9 или выше
· Установленный и запущенный Redis (например, redis-server)
· Telegram Bot Token (получить у @BotFather)
· Telegram канал (для публикаций), в который бот добавлен как администратор
· Telegram группа (опционально), куда бот будет постить все новости

Шаги по установке

1. Клонируйте репозиторий
   ```bash
   git clone https://github.com/yourusername/ultra-crypto-bot.git
   cd ultra-crypto-bot
   ```
2. Создайте виртуальное окружение и активируйте его
   ```bash
   python -m venv venv
   source venv/bin/activate      # Linux/Mac
   venv\Scripts\activate         # Windows
   ```
3. Установите зависимости
   ```bash
   pip install -r requirements.txt
   ```
4. Создайте файл .env в корне проекта и заполните его (см. раздел «Конфигурация» ниже)
5. Убедитесь, что Redis запущен
   ```bash
   redis-server
   ```
6. Запустите бота
   ```bash
   python bot.py
   ```
7. (Опционально) Запустите веб‑интерфейс в отдельном терминале
   ```bash
   uvicorn web:app --reload --port 8000
   ```
   После этого веб‑дашборд будет доступен по адресу http://localhost:8000

⚙️ Конфигурация (файл .env)

Создайте файл .env в корне проекта и укажите следующие переменные:

```ini
BOT_TOKEN=ваш_токен_бота
CHANNEL_ID=@имя_канала_или_id
GROUP_CHAT_ID=-1001234567890        # ID группы (обязательно с минусом)
ADMIN_IDS=123456789,987654321       # ID администраторов через запятую
REDIS_URL=redis://localhost:6379/0

# Настройки определения «триггерных» новостей
TRIGGER_PRICE_CHANGE_PERCENT=2.0    # минимальное изменение цены в %
TRIGGER_TIMEFRAME_MINUTES=30        # временной интервал для проверки (в минутах)
```

Примечание:

· CHANNEL_ID может быть либо username канала (например, @mycryptochannel), либо его числовой ID (начинается с -100).
· GROUP_CHAT_ID также должен начинаться с -100.
· ADMIN_IDS — список числовых ID пользователей Telegram, которые получат доступ к админ‑командам.

🌐 Веб‑интерфейс

Веб‑дашборд доступен по адресу http://localhost:8000 (если запущен). Он показывает:

· Общее количество новостей в базе данных
· Количество триггерных новостей (совпавших с рынком)
· Количество новостей, опубликованных в канале (на данный момент равно числу триггерных)

Вы можете легко расширить его, добавив графики и более детальную аналитику.

🧠 Как работает определение «триггерных» новостей

1. Бот получает новость из SSE‑потока или периодически запрашивает последние новости.
2. Сохраняет новость в БД вместе с текущим временем.
3. Запрашивает тональность новости через эндпоинт /api/ai/sentiment.
4. Через заданный интервал (TRIGGER_TIMEFRAME_MINUTES) проверяет изменение цены указанной монеты (через внешний API исторических цен, например CoinGecko). Если изменение превышает TRIGGER_PRICE_CHANGE_PERCENT и совпадает по направлению с тональностью, новость считается триггерной.
5. Триггерные новости публикуются в канал.

Важно: в текущей реализации используется заглушка для изменения цены (случайное значение). Для работы в реальных условиях необходимо интегрировать API исторических цен (например, CoinGecko) и заменить функцию _fetch_price_change в services/trigger_detector.py.
